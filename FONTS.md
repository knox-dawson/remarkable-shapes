# Fonts

This project uses vector stroke fonts rendered through `Remarkable::LineFont`.

- `line_font`

- `line_font_cursive`
- `line_font_mono`

- `relief_singleline_italic`

## YAML Usage

Use `font:` to select a family:

```yaml
objects:
  - type: text
    x: 170
    y: 240
    width: 1000
    height: 60
    text: "Synthetic italic sample"
    size: 36
    stroke_width: 3
    font: line_font_italic
    color: black
```

Relief example:

```yaml
objects:
  - type: text
    x: 170
    y: 320
    width: 1000
    height: 60
    text: "Relief italic sample"
    size: 36
    stroke_width: 3
    font: relief_singleline_italic
    color: black
```

## Ruby Usage

```ruby
Remarkable::Shapes.text(
  page,
  "Hello",
  170,
  240,
  size: 36,
  stroke_width: 3,
  font: :line_font_italic
)
```

## Temporary Compatibility

The older text options are still accepted temporarily during the current beta transition:

- `style: italic` maps to `line_font_italic`
- `style: cursive` maps to `line_font_cursive`
- `mono: true` maps to `line_font_mono`

These compatibility paths are marked with TODO comments in the code and are intended to be removed after the next beta release.

## Synthetic Italics

`line_font_italic` and `relief_singleline_italic` are generated by slanting the plain glyph coordinates.

The built-in synthetic italic family also has a small pair-adjustment table for spacing corrections in combinations such as:

- `ST`
- `TU`
- `UV`
- `VW`
- `WX`
- `XY`
- `YZ`
- `12`
- `78`
- `ef`
- `st`
- `ij`
- `wx`

The current implementation lives in [lib/shapes/line_font.rb](/home/bmb/rmlines_research/remarkable-shapes/lib/shapes/line_font.rb:1).

## Conversion Tools

Convert a UFO source into the line-font JSON stroke format:

```bash
bin/ufo_to_line_font --ufo PATH_TO_FONT.ufo --output data/my_font/plain.json
```

Generate one full-coverage sample page per font family:

```bash
bin/generate_font_samples
```

This writes:

- `examples/font-page-*.yml`
- `out/font-page-*.rmdoc`
